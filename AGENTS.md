# AGENTS.md

## 教程写作规范

- 标题要简洁，优先让读者一眼看懂主题。
- 正文少 AI 味，避免空话、套话、翻译腔和过度解释。
- 使用中文文章标点，中文段落中不要混用英文逗号、句号。
- 段落保持紧凑，普通段落之间尽量少留空行；标题、图片、表格、列表和代码块前后保留必要空行。
- 文章以实用为主，减少重复介绍；同一概念已经在其他文档讲过时，只保留必要引用和核心结论。
- 删除填充短语，少用“可以理解为”“简单说”“需要注意的是”等解释腔。
- 避免金句式结尾，结尾优先给具体动作或使用场景。
- 少用三段式排比，能用两项或四项就不要强行凑三项。
- 长句优先拆短，具体信息优先于抽象判断。
- 图片较多时，减少重复表格和重复说明。

## 图片规范

- 配图优先使用官方图标、官方界面图或用户提供的真实素材。
- 没有官方素材时，再做自定义设计图。
- 自定义图要保持中文可读、颜色对比清晰、布局不遮挡文字。
- 提交前检查所有 Markdown 图片引用，确保文件存在且显示位置合理。

## 提交前检查

- 检查 Markdown 链接、图片引用和代码块是否完整。
- 检查是否存在乱码、明显错别字和过时文件名。
- README 有新增文章或素材时，需要同步更新目录和图片说明。

## 文章编号与上传

- 新文章使用 `NN-中文标题.md` 命名，但不要再放在仓库根目录；必须放入对应中文分类目录（如 `01-工具教程/`、`02-AI知识/`、`03-好文方法/`）。发布前检查 `00-知识库导航/知识库总索引.md`，使用下一个未占用的两位数编号；不得重编号已发布文章。
- 新增配图必须使用同一篇文章的 `NN-` 文件名前缀；封面放入 `08-素材库/图片/文章封面/`，正文插图放入 `08-素材库/图片/正文插图/`，并同步更新 README 与知识库索引。
- 新增文章时同步更新 `README.md`、`00-知识库导航/知识库总索引.md`、`分类索引.md`、`系列索引.md`、`标签索引.md` 和 `07-资料与流程/发布记录.md`；目录、引用图片和文件名均按编号升序排列。
- 除非任务明确要求“仅草稿”或“不上传”，文章和素材完成本地校验后提交并推送 `main`；推送前先获取远端更新，禁止强推。


## 知识库目录约定

- 顶层只保留 GitHub/Codex 必需入口文件（`README.md`、`AGENTS.md`）和中文一级目录。
- 文章按主分类进入中文目录；交叉分类通过索引和标签解决，不复制正文文件。
- 素材、脚本、流程文档分别放入 `08-素材库/`、`09-工具脚本/`、`07-资料与流程/`；不保留整包发布归档，只保留正式文章实际引用的图片素材。
