## 一、当前内容基础

当前知识库已经形成三条主线：

1. **工作流与交付**：从可交付工作流、一人 AI 工作台、内容工作流、验收标准，到从会提问升级到会安排 AI 干活。
2. **工具与 Agent**：从 Codex、Skill、MCP、Computer Use，到 AI Coding、Agent 工作流、多 Agent 系统和搜索 Agent 化。
3. **安全与边界**：从数据边界、电脑操作权限、Agent 权限，到个人与团队的验收清单。

后续选题不再重复讲工具清单，而是补齐一条更适合普通读者连续学习的路径：

```text
认知入门 → 工具选择 → 提问表达 → 任务拆解 → 资料组织 → 结果验收 → 模板复用 → 场景实战 → 自动化 → 安全收口
```

## 二、后续系列定位

系列名称建议：**普通人 AI 学习教程：从会问到会交付**

目标读者：

- 刚开始学习 AI，但被工具、模型和 Prompt 清单淹没的普通职场人。
- 已经用过 AI 写文、做 PPT、查资料，但输出质量不稳定的人。
- 想把 AI 变成稳定工作方法，而不是偶尔试用新工具的人。

写作原则：

- 每篇只解决一个具体问题，避免泛泛讲概念。
- 每篇都必须给读者一个可复用产物：清单、模板、步骤、评分表或练习任务。
- 文章顺序由浅入深，不把自动化、Agent、MCP 放在新手第一步。
- 与既有文章互相衔接，但标题和正文避免成为旧文换皮。

## 五、单篇写作大纲

### 35 普通人学 AI，先把问题变成任务

- 开头：多数人不是不会用 AI，而是不知道把真实问题交代成任务。
- 核心观点：学 AI 的第一步不是背 Prompt，而是把模糊需求改写成可执行任务。
- 正文结构：
  1. 普通人学习 AI 的三个误区：追工具、抄提示词、求万能答案。
  2. 一个任务至少包含目标、背景、约束、交付物、验收标准。
  3. 用三个日常例子演示：写总结、学课程、做汇报。
  4. 给读者一张任务改写练习表。
- 交付物：任务定义卡。
- 衔接：下一篇进入工具选择，不再盲目注册工具。

### 36 别再追工具榜单，一人最小 AI 工具箱怎么搭

- 开头：工具越多，普通人越容易不知道从哪里开始。
- 核心观点：工具选择应按任务链路，而不是按热度榜单。
- 正文结构：
  1. 工具箱的四类角色：主模型、搜索研究、文件处理、自动化执行。
  2. 选择工具的五个指标：稳定、成本、隐私、可导出、可复用。
  3. 新手最小组合：一个主对话工具 + 一个资料工具 + 一个文档或表格工具。
  4. 什么情况下才需要 Agent、MCP 或自动化。
- 交付物：一人 AI 工具箱选择表。
- 衔接：工具选好后，进入需求表达。

### 37 会提问不是会聊天，把需求说清楚的四层结构

- 开头：很多失败输出不是 AI 不行，而是需求只说了一半。
- 核心观点：有效提问不是写神奇咒语，而是补齐任务信息。
- 正文结构：
  1. 第一层：我要完成什么目标。
  2. 第二层：我有哪些背景和资料。
  3. 第三层：有哪些限制和偏好。
  4. 第四层：输出要长什么样，怎样算合格。
  5. 同一个需求从一句话改到可执行 Prompt 的过程。
- 交付物：四层需求模板。
- 衔接：需求说清后，再拆成步骤。

### 38 一个大任务怎么拆给 AI，先拆结果再拆步骤

- 开头：直接把大任务扔给 AI，最容易得到完整但不可用的答案。
- 核心观点：拆任务要先拆交付结果，再拆执行步骤。
- 正文结构：
  1. 大任务失败的三种表现：跑题、漏项、无法验收。
  2. 拆解顺序：目标结果 → 中间产物 → 每步输入 → 每步验收。
  3. 三个例子：做汇报、写长文、整理学习笔记。
  4. 什么时候让 AI 先出计划，什么时候直接执行。
- 交付物：任务拆解表。
- 衔接：任务拆开后，需要给每一步准备资料。

### 39 给 AI 喂资料不是越多越好，上下文包实操清单

- 开头：把所有资料都丢给 AI，常常等于把噪音也一起交给它。
- 核心观点：上下文包的价值在于筛选、排序和说明，而不是堆文件。
- 正文结构：
  1. AI 需要的不是所有信息，而是与任务有关的信息。
  2. 上下文包四件套：目标、资料、规则、样例。
  3. 怎么删掉无关资料、过期资料和重复资料。
  4. 用一个工作汇报场景演示上下文包。
- 交付物：上下文包模板。
- 衔接：资料准备好后，重点转向结果验收。

### 40 AI 答案怎么验收，事实、逻辑、格式、风险四步检查

- 开头：AI 生成得快，不代表你交付得快。
- 核心观点：不会验收，AI 只会把错误更快地送到你面前。
- 正文结构：
  1. 事实检查：有没有编造、过时、无来源。
  2. 逻辑检查：结论是否由材料推出。
  3. 格式检查：是否符合交付要求。
  4. 风险检查：隐私、版权、合规、误导性表达。
  5. 给出不同场景的最低验收标准。
- 交付物：四步验收清单。
- 衔接：验收标准稳定后，沉淀成模板库。

### 41 把常用 AI 方法沉淀成个人模板库

- 开头：真正省时间的不是一次好答案，而是下次能复用的方法。
- 核心观点：模板库是普通人把 AI 能力稳定下来的第一层资产。
- 正文结构：
  1. 哪些任务值得模板化。
  2. 一个模板至少包含：适用场景、输入材料、步骤、输出格式、验收标准。
  3. Prompt、模板、Skill 的区别。
  4. 如何每次用完做一次小版本更新。
- 交付物：个人 AI 模板库目录结构。
- 衔接：模板库建好后，进入真实工作场景。

### 42 用 AI 做真实工作，办公、写作、学习、数据四个场景

- 开头：普通人学 AI，最好从今天就要完成的任务开始。
- 核心观点：AI 学习不应脱离真实交付物。
- 正文结构：
  1. 办公场景：会议纪要、周报、汇报提纲。
  2. 写作场景：选题、提纲、初稿、改写。
  3. 学习场景：课程拆解、错题复盘、读书笔记。
  4. 数据场景：表格清洗、趋势描述、结论提炼。
  5. 每个场景都给一个任务卡和验收标准。
- 交付物：四类场景任务卡。
- 衔接：当任务重复出现，再考虑自动化。

### 43 AI 自动化入门，哪些流程适合交给工具接管

- 开头：不是所有重复动作都值得自动化。
- 核心观点：自动化前先确认流程稳定、边界清楚、失败可回滚。
- 正文结构：
  1. 适合自动化的流程：频繁、稳定、低风险、有明确输入输出。
  2. 不适合自动化的流程：判断复杂、权限敏感、后果不可逆。
  3. MCP、Computer Use、脚本自动化分别适合什么。
  4. 人工确认点怎么设置。
- 交付物：自动化适配度评分表。
- 衔接：自动化越多，越需要安全收口。

### 44 普通人使用 AI 的安全收口，权限、隐私、成本和审计

- 开头：AI 越能帮你做事，越要知道它不能碰什么。
- 核心观点：安全不是不用 AI，而是给 AI 设置清晰边界。
- 正文结构：
  1. 权限边界：账号、文件、系统操作权限。
  2. 隐私边界：个人信息、客户信息、未公开资料。
  3. 成本边界：订阅、API、时间返工成本。
  4. 审计边界：过程记录、人工确认、可回滚。
  5. 给个人和小团队两版清单。
- 交付物：AI 安全收口清单。
- 衔接：作为本轮教程系列的收束篇，也可反向链接到 19、24、29 号文章。

## 六、更新规则

1. 每启动一篇文章，把总表的 `当前阶段` 从 S0 改为 S1，并在 `产物状态` 填写 Brief 文件或正式文章路径。
2. 每完成一个阶段，只更新本文件对应行，不删除历史记录。
3. 如果选题被合并到其他文章，把阶段改为 DROP，并在 `下一步` 写明合并到哪一篇。
4. 如果等待最新工具、平台规则或官方资料，把阶段改为 HOLD，并写明恢复条件。
5. 正式入库后，同步更新总索引、分类索引、系列索引、标签索引和发布记录。
6. GitHub 推送完成后，在下方追加一条状态变更记录，写明日期、文章编号、阶段、提交哈希和待观察项。
