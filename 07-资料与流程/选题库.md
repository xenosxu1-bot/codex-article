> 更新时间：2026-07-17
> 作用：集中管理后续文章选题，避免与已发布内容重复，并把灵感沉淀为可排期、可写作、可质检、可入库的内容资产。
> 当前保留正式文章：19 篇；已下架编号：12、13、15、20、22；编号不重排，以 `文章资产登记表.md` 和 `选题文章绑定表.md` 为准。

## 一、使用原则

1. **先查重，再立题**：新增选题前，先查 `选题文章绑定表.md`、`已发布文章去重登记.md`、`文章资产登记表.md` 和本文档，确认不是旧文换皮。
2. **先交付物，再标题**：每个选题必须说明读者最后能拿走什么，例如清单、模板、流程图、评分表、案例复盘。
3. **先系列位置，再写正文**：选题必须归入一个系列，避免单篇文章互相孤立。
4. **先风险分级，再做资料研究**：热点、产品、法律、安全、价格、模型能力等容易变化的信息，必须重新核查公开资料；纯经验方法也要标明是经验判断。
5. **先进入 main，确认后再同步 prod**：`main` 作为个人规划与生产工作区，`prod` 只接收确认可公开共享的知识库内容。

## 四、状态流转

```text
S0 灵感池 → S1 已立题 → S2 已做资料/Brief → S3 初稿中 → S4 待质检 → S5 待发布 → S6 已入库 → S7 已复盘
                         ↘ HOLD 暂缓 / DROP 放弃
```

- `S0`：只有题目或问题，尚未确认是否值得写。
- `S1`：已完成查重、分类、系列和交付物定义。
- `S2`：资料边界、事实核查点、结构 Brief 已准备。
- `S3`：正文生产中。
- `S4`：进入文章质量检查，包括格式、脚注、图片版权、事实边界。
- `S5`：等待发布或同步。
- `S6`：已写入资产登记表、导航索引和发布记录。
- `S7`：发布后根据阅读、反馈、复用价值完成复盘。

## 五、优先级评分

每个维度 1—5 分，总分用于决定优先级：

| 维度 | 判断问题 |
| --- | --- |
| 读者痛点 | 读者是否会主动搜索、收藏或转发？ |
| 可操作性 | 是否能给出步骤、模板、清单或案例？ |
| 差异化 | 是否不是旧文换标题？是否有新场景、新证据或新交付物？ |
| 系列价值 | 是否能补齐现有知识库路径？ |
| 更新价值 | 是否对后续文章、Skill、工具脚本或知识库维护有复用价值？ |

- 22—25 分：P0，本月优先推进。
- 18—21 分：P1，作为近期排期候选。
- 14—17 分：P2，进入储备观察。
- 14 分以下：暂不写，除非后续出现强资料或热点机会。

## 七、选题与生成文件绑定

- 绑定表：`选题文章绑定表.md`，用于把选题ID、正式文章编号和正文路径连起来。
- 刷新命令：`python 09-工具脚本/选题文章绑定检查.py`。
- 当前结论：`T075` 已绑定正式文章 26；`T059` 与正式文章 16 疑似重复/可合并；29 条选题的 `关联已发布` 编号含失效项，写作前需要校准。
- 使用规则：看到“已生成”时只做修订或更新；看到“疑似重复/可合并”时先改成新角度；看到“关联旧文”时在正文里明确新增价值。

## 八、每次新增选题的固定动作

1. 在本文档新增一行候选选题。
2. 到 `选题文章绑定表.md` 和 `已发布文章去重登记.md` 检查标题、关键词、核心问题和交付物是否重叠。
3. 如果与旧文相近，必须明确“新增角度”：新场景、新读者、新案例、新模板或新资料。
4. 如果进入写作，先补 Brief，再进入正文，不直接让 AI 按标题扩写。
5. 发布后把正式编号、标题、文件、分类、标签、封面和状态写入 `文章资产登记表.md`，并同步更新本文档状态。
6. 运行 `python 09-工具脚本/选题文章绑定检查.py` 刷新绑定表，确认新文章不再被误判为待生成。
