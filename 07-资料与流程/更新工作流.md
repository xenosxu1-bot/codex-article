# 更新工作流

> 更新时间：2026-07-15
> 本文件是仓库维护时的简版流程；完整文章生产标准见 [后续文章生成方案](后续文章生成方案.md)。

## 1. 新增文章前

1. 明确文章类型：工具教程、AI知识、好文方法、安全治理、案例实战、热点追踪。
2. 检查 [知识库总索引](../00-知识库导航/知识库总索引.md)，确认下一个未占用编号。
3. 明确是否需要联网核验：最新模型、产品、价格、政策、论文、新闻和平台规则必须核验。
4. 明确图片需求：封面放入 `08-素材库/图片/文章封面/`，正文插图放入 `08-素材库/图片/正文插图/`。

## 2. 文件命名

- 文章：放入对应中文分类目录，文件名使用 `NN-中文标题.md`。
- 封面：`08-素材库/图片/文章封面/NN-中文文章标题-封面.jpg`。
- 正文插图：`08-素材库/图片/正文插图/NN-中文文章标题-正文插图01.png`。
- 脚本：放入 `09-工具脚本/`，文件名尽量中文化并说明用途。
- 发布包：不长期保留整包归档；只保留正式文章实际引用的图片素材。

## 3. 文章格式

每篇正式文章顶部使用隐藏元信息块，避免 GitHub/Markdown 预览展示元信息表格：

```markdown
<!--
---
id: "35"
title: "文章标题"
description: "一句话说明文章解决什么问题"
author: "Codex Article Lab"
published: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
category: "工具教程/AI知识/好文方法/安全治理/案例实战/热点追踪"
series: "所属专题系列"
tags: ["标签1", "标签2"]
status: "draft/review/published/outdated"
reading_time: "6 分钟"
source_status: "已核验/待核验/本地经验"
cover: "../08-素材库/图片/文章封面/NN-中文文章标题-封面.jpg"
---
-->
```

正文不要再次写 `# 文章标题`，从核心观点、封面图或 `##` 小标题开始。

## 4. 发布前检查

- 图片链接是否都能找到。
- 本地 Markdown 链接是否都能打开。
- 是否存在乱码、问号占位符或未闭合代码块。
- 正文是否隐藏元信息，且没有重复一级标题。
- README、知识库总索引、分类索引、系列索引、标签索引是否同步。
- 发布记录是否新增一条。

## 5. 推荐协作方式

- 重复写作流程：沉淀到 `article-wechat` Skill。
- 项目长期规则：写入 `AGENTS.md`。
- 需要外部资料：优先使用浏览器、MCP、插件或官方文档连接器。
- 想定期追踪热点：适合做成自动化任务，每周汇总候选选题。
