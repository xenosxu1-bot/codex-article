# 更新工作流

> 更新时间：2026-07-15

本仓库采用“文章入库 + 分类 + 索引 + 发布记录”的维护方式，并已固化为 article-wechat Skill 的文章知识库维护流程。

## 默认流程

1. 明确文章类型、目标读者、资料边界和是否发布。
2. 根据 `后续文章生成方案.md` 先形成 Brief。
3. 生成或修订正文：正文不放顶部元信息块，不重复一级标题。
4. 将文章编号、标题、摘要、分类、标签、封面、状态等写入 `文章资产登记表.md`。
5. 图片进入 `08-素材库/图片/文章封面/` 或 `08-素材库/图片/正文插图/`。
6. 同步更新总索引、分类索引、标签索引、系列索引和分类 README。
7. 更新发布记录和内容维护清单。
8. 检查 Markdown 链接、图片链接和文章格式。
9. 用户明确要求发布、提交、推送或同步 GitHub 时，再提交并推送。

## 正文格式约定

- 文章文件名承接正式标题。
- 正文不再写 `# 正式标题`。
- 正文顶部不写 YAML frontmatter。
- 正文顶部不写 HTML 注释元信息。
- 可从“副标题：”、封面图、导语或 `## 开篇` 开始。

## 元信息维护约定

正式文章的维护字段统一外置到：

- `07-资料与流程/文章资产登记表.md`

索引、发布记录、后续批量维护优先以该登记表为准。

## 知识库导航分层规则

- 分类是主书架：每篇文章只选一个主分类，并与正文所在目录保持一致。
- 系列是阅读路径：可以跨分类组织文章，用于新手路线、专题学习和项目复盘。
- 标签是检索词：每篇文章建议 3-5 个，优先使用稳定、可复用、可跨分类的关键词。
- 总索引是全量清单：用于确认编号、路径、分类、系列、标签和状态。
- 文章资产登记表是唯一元数据来源：分类索引、系列索引、标签索引和总索引都以它为准同步维护。
- 读者入口优先展示首页、系列和分类；标签和登记表作为辅助检索与后台维护入口。
