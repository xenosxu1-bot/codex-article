# 更新工作流

> 更新时间：2026-07-15

这份工作流用于后续持续更新知识库，目标是让每次新增文章都有固定入口、固定分类、固定检查方式。

## 1. 新增文章前

1. 明确文章类型：工具教程、AI知识、好文方法、安全治理、案例实战、热点追踪。
2. 检查 [知识库总索引](../00-知识库导航/知识库总索引.md)，确认下一个未占用编号。
3. 如果文章涉及最新模型、产品、价格、政策或官方规则，先核对官方或可信来源。
4. 确定配图需求，图片统一放入 `08-素材库/图片/`。

## 2. 文件命名

- 文章：放入对应分类目录，文件名使用 `NN-中文标题.md`。
- 图片：放入 `08-素材库/图片/`，建议使用 `NN-cover.jpg`、`NN-inline-01.jpg`。
- 发布包：放入 `08-素材库/发布包/`。
- 脚本：放入 `09-工具脚本/`，文件名尽量中文化并说明用途。

## 3. 文章元数据

每篇文章顶部建议保留以下字段：

```yaml
---
id: "35"
title: "文章标题"
description: "一句话说明文章解决什么问题"
author: "Codex Article Lab"
published: "YYYY-MM-DD"
updated: "YYYY-MM-DD"
category: "工具教程/AI知识/好文方法/安全治理/案例实战/热点追踪"
series: "所属专题系列"
tags: ["标签1", "标签2"]
status: "draft/review/published/outdated"
reading_time: "6 分钟"
source_status: "已核验/待核验/本地经验"
cover: "../08-素材库/图片/NN-cover.jpg"
---
```

## 4. 发布前检查

- 图片链接是否都能找到。
- 本地 Markdown 链接是否都能打开。
- 是否存在乱码、问号占位符或未闭合代码块。
- README、知识库总索引、分类索引、系列索引、标签索引是否同步。
- 发布记录是否新增一条。

## 5. 推荐协作方式

- 重复写作流程：沉淀到 `article-wechat` Skill。
- 需要外部资料：优先使用浏览器、MCP、插件或官方文档连接器。
- 想定期追踪热点：适合做成自动化任务，每周汇总候选选题。
