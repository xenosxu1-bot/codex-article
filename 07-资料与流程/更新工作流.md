# 更新工作流

> 更新时间：2026-07-20

本仓库采用“文章入库 + 分类 + 索引 + 发布记录”的维护方式，并已固化为 article-wechat Skill 的文章知识库维护流程。

## 默认流程

1. 明确文章类型、目标读者、资料边界和是否发布。
2. 根据 `后续文章生成方案.md` 先形成 Brief。
3. 生成或修订正文：正文不放顶部元信息块，不重复一级标题。
4. 将文章编号、标题、摘要、分类、标签、封面、状态等写入 `文章资产登记表.md`。
5. 图片进入 `08-素材库/图片/文章封面/` 或 `08-素材库/图片/正文插图/`。
6. 同步更新总索引、分类索引、标签索引、系列索引和分类 README。
7. 更新发布记录和内容维护清单。
8. 检查 Markdown 链接、图片链接和文章格式。
9. 用户明确要求发布、提交、推送或同步 GitHub 时，再提交并推送。

## 正文格式约定

- 文章文件名承接正式标题。
- 正文不再写 `# 正式标题`。
- 正文顶部不写 YAML frontmatter。
- 正文顶部不写 HTML 注释元信息。
- 可从“副标题：”、封面图、导语或 `## 开篇` 开始。

## 元信息维护与唯一事实来源

正文只承载读者可见内容；维护字段统一外置，避免同一信息在多份文档中被手工重复修改。

| 信息 | 唯一事实来源 | 其他文件如何使用 |
| --- | --- | --- |
| 正式文章编号、标题、分类、系列、标签、路径、字数、入库状态 | `文章资产登记表.md` | README、首页、总索引、分类/系列/标签索引由重建脚本读取或对照该表 |
| 选题阶段、目标读者、差异化角度、暂停或放弃原因 | `选题库.md` | `选题文章绑定表.md` 仅用于生成映射和发现重复 |
| 已发布主题的去重结论 | `已发布文章去重登记.md` | 新选题立项前必须对照，不反向覆盖选题库 |
| GitHub 提交、入库和发布动作 | `发布记录.md` | 只记录发生过的动作和范围，不重复维护文章基础字段 |
| 图片生成方式、来源、版权或授权核验 | `图片生成记录.md` | 图片文件名、文章编号和资产登记表共同构成追溯线索 |
| 下架原因、替代内容、素材清理和旧号追溯 | `下架文章与替代关系.md`、`编号变更记录.md` | 正式编号立即连续补位；旧号只保留在追溯记录中，不作为活跃编号使用 |

新增或修订文章时，先更新对应的唯一事实来源，再运行索引、绑定和图片检查；不要直接在多个索引文件中手工改同一项标题、状态或路径。

## 一键发布前检查

为减少手工串联检查的时间，常规维护、文章入库、索引更新和推送前优先运行：

```powershell
python 09-工具脚本/一键发布检查.py
```

该脚本默认只检查和重建必要索引，不自动提交、不自动推送。检查范围包括：

1. 重建 README、知识库首页、总索引、分类索引、系列索引、标签索引和分类 README。
2. 执行文章质量扫描，确认 P0/P1 清零。
3. 刷新选题文章绑定表，确认失效关联为 0。
4. 检查图片资产：正文图片引用必须存在，正式文章应有同编号封面，历史未引用图片只作为 P2 提醒。
5. 检查全库 Markdown 本地链接和图片引用。
6. 检查正式文章文件数、资产登记表和 README 展示数量一致，README 展示编号与文章文件名前缀一致。
7. 执行 Git 格式检查、删除项检查和敏感信息风险检查。

只有脚本通过后，才进入提交与推送；如果脚本提示 P2 历史素材提醒，可在不阻断发布的前提下另开清理任务。

## 下架与素材处理

1. 在 `下架文章与替代关系.md` 登记原编号、原因、替代内容和素材处理结果。
2. 清理正文、索引和登记表中的正式引用。
3. 运行 `09-工具脚本/图片资产检查.py`，处理已下架编号残留的封面或正文插图。
4. 运行 `09-工具脚本/重建知识库索引.py`，确认读者入口中不再出现旧链接。

## 选题库与去重管控

新增或重写文章前，必须先完成以下动作：

1. 在 `选题库.md` 登记选题，明确分类、系列、目标读者、核心问题和交付物。
2. 在 `已发布文章去重登记.md` 检查 01—当前最大编号文章是否已经覆盖同类主题。
3. 如果主题高重叠，必须写明新增角度，例如新场景、新模板、新案例、新资料或团队版，而不是旧文换标题。
4. 进入正文前先形成 Brief，不直接按标题扩写。
5. 发布或入库后，同步更新 `文章资产登记表.md`、`发布记录.md`、导航索引、选题库状态和去重登记。
6. 分支同步遵循 `选题同步与规划管控机制.md`：默认先改 main，用户确认后再同步 prod。
## 编号重排

- 删除或下架后，必须在同一维护批次内从首个缺号开始连续补位：先执行 `python 09-工具脚本/reindex_formal_articles.py`，再更新映射与索引，最后运行 `python 09-工具脚本/一键发布检查.py`。
- 执行前建立旧号→新号映射；同步正文、正在使用图片、元数据、资产登记表、README/索引、本地链接和发布记录。日常发布检查不运行文件重命名，避免无删除操作改变正式文章路径。
- 历史发布记录保留原编号，以 `编号变更记录.md` 追溯。
