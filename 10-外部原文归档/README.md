## 文件约定

- `source.html`：下载时的原始 HTML 快照，是内容追溯的唯一原文，禁止人工编辑。
- `content.md`：由原始 HTML 自动转换得到的 Markdown，只转换格式，不人工改写正文。
- `content.txt`：规范化纯文本，仅供检索、指纹计算和重复度检查。
- `metadata.json`：来源、权属、抓取日期、哈希和去重规则。

## 同步账号原创文章

1. 在已登录的公众号后台进入“内容管理 → 原创 → 文章原创”，导出不含后台 token 的公开文章 URL 清单。
2. 将清单保存为 JSON（每项至少含 `title`、`url`、`published_at`），然后运行：

```powershell
node 09-工具脚本/批量归档微信公众号原创.mjs `
  --manifest 10-外部原文归档/微信原创文章清单-YYYY-MM-DD.json `
  --captured-at YYYY-MM-DD `
  --report 10-外部原文归档/微信原创文章同步记录-YYYY-MM-DD.json
```

3. 运行 `node 09-工具脚本/重建内容库索引.mjs` 更新统一内容库索引。
4. 核对同步记录中的 `failed` 为 0，再提交原始快照、转换文本、元数据、来源清单和同步记录。

单篇补录仍可使用 `node 09-工具脚本/归档外部原文.mjs`。当传入 `--rights account-owner-authorized` 时，元数据会明确标注为账号后台授权的原创原文。

## 查重规则

- 所有外部原文快照都应保留为基线，**即使它与既有正式文章重复度达到或超过 50%**；这是追溯关系，不是可发布的新稿。
- 新增或重写正式文章前，运行 `node 09-工具脚本/检查内容重复度.mjs --candidate <候选文件>`。候选稿相对每一条基线的重复度必须 **低于 50%**；规范化标题或正文完全一致也会直接阻止。
- 重复度 30%–49.99% 为人工复核；必须在选题记录中说明新的读者场景、核心问题和交付物。
