# 3分钟看懂 WorkBuddy：腾讯正在打造真正的“AI员工”

WorkBuddy 是腾讯推出的全场景职场 AI 智能体桌面工作台。
与传统的 AI 聊天工具最大的区别在于——它不仅会“说话”，更会“干活”。
你只需用一句话描述需求，它就能像同事一样自主拆解任务、规划步骤、执行操作，最终交付可以直接使用的工作成果（文档、表格、PPT 等），而不是仅仅给你一堆建议。
本文将从安装配置到进阶玩法，手把手带你把 WorkBuddy 变成真正能落地的 AI 同事。

1. 安装与登录
• 下载安装：访问 workbuddy.tencent.com，选择对应系统版本直接运行安装。

• 登录授权：使用微信、企业微信或 QQ 账号扫码登录。首次登录时建议仅授权工作目录，以保障本地数据安全。

3. 第一个任务
打开 WorkBuddy 后，在输入框中用一句话描述你的需求，例如：
• “帮我把这份销售数据生成一个分析报告”

• “整理这个文件夹里的图片，按日期重命名”

• “根据这份需求文档生成一个 PPT”

点击发送或按回车键，WorkBuddy 就会开始自主执行。整个过程无需额外操作，你可以在右侧结果区实时查看产物和变更。

对话中的常见操作
在任务执行过程中，你可以随时：
• 继续追问：直接在输入框输入新消息，让 WorkBuddy 基于已有的上下文继续处理

• 中断任务：任务正在执行时，输入框区域会显示停止按钮，可随时中断

• 保留上下文：使用顶部“保留工作空间”按钮，在同一个项目中持续追问，减少重复切换

顶部操作栏还提供了 “打开编辑器” 按钮，方便你一边看对话结果，一边回到代码或文件继续处理。

• 粘贴截图：Ctrl/Cmd + V 直接粘贴剪贴板中的图片，WorkBuddy 能识别其中的文字和内容

• 拖拽文件：直接把文件拖到输入框区域

• 混合输入：结合文件、图片和文字说明，让 AI 更全面地理解问题

小技巧：如果你需要在已有任务中补充新的文件上下文，直接拖拽即可，无需重新创建任务。
4.核心能力详解
本地文件操作
WorkBuddy 获得了本地文件夹的授权后，可以自动完成文件读写、批量处理等操作：
• 批量重命名、整理文件夹

• 读写 Word、Excel、PPT、PDF 等办公文档

• 图片处理和格式转换

• 数据文件的分析与可视化

所有操作都在沙箱隔离环境中执行，数据不上传云端，高危指令可被拦截。
注意：首次使用涉及本地文件的功能时，WorkBuddy 会请求授权，建议仅授权必要的工作目录。
多任务并行
WorkBuddy 支持同时创建和管理多个任务，多个智能体可以并行处理不同类型的工作：
• 一边让 AI 生成报告，另一边处理数据分析

• 左侧任务列表中可以随时切换查看不同任务的进度

• 适合需要同时处理多个工作场景的用户

模型自由切换
WorkBuddy 内置了混元、DeepSeek、GLM、Kimi 等 5+ 款主流大模型，可以在界面中直接切换：
• 不同任务选择不同的模型，按需匹配

• 每日有免费 Credits 额度，新用户上线期间可领取 5000 Credits 福利

• 支持自定义接入 OpenAI 兼容格式的任意模型

手机端远程调度
WorkBuddy 支持通过企微、QQ、飞书、钉钉等通讯工具远程“遥控”电脑执行任务：
配置方法：进入“个人中心 → Claw 设置”，选择你要绑定的 IM 工具，按照引导完成配置。配置完成后，即使人不在电脑旁，也能通过手机端的对应应用向 WorkBuddy 下达任务。
使用场景示例：出差途中用手机让电脑自动完成社交媒体内容的撰写与发布，或远程触发定时报告的生成与发送。
5.进阶玩法
自定义 Skills 技能包
WorkBuddy 预置了 20+ 专业技能包，覆盖数据分析、内容创作、自动化办公等常见场景。更重要的是，它支持零代码自定义 Skills：

• 导入官方技能：从技能市场一键安装，像安装手机 App 一样简单

• 自定义新建：用 SKILL.md 格式封装你的专属工作流，将特定领域的经验和操作流程固化下来

• 共享给团队：自定义的 Skills 可以在团队内共享，形成统一的工作规范

Skill 的本质是一套“操作说明书”，你可以把复杂的多步骤任务（如“每周五自动生成销售周报并发送邮件”）打包成一个指令，后续直接一键调用。
自定义 API 接入（实现“模型自由”）
如果你想使用 WorkBuddy 内置之外的模型，可以通过自定义 API 接入任意 OpenAI 兼容的大模型：
配置步骤：
1. 在用户目录下创建配置文件夹：

◦ macOS / Linux：mkdir -p ~/.workbuddy

◦ Windows：在 %USERPROFILE%\.workbuddy 创建文件夹

2. 在文件夹中新建 models.json 文件，写入以下结构（可同时配置多个模型）：

{
"models":[
{
"id":"your-model-id",
"name":"your-model-name",
"vendor":"YourVendor",
"url":"https://your-api-endpoint/v1/chat/completions",
"apiKey":"your-api-key",
"maxInputTokens":128000,
"maxOutputTokens":4096
}
]
}
3. 保存后在 WorkBuddy 的设置中选择你添加的自定义模型即可使用。

适用场景：当内置模型的免费额度不够用、或你想接入特定领域的专业模型时，这项功能非常实用。
自动化工作流与定时任务
WorkBuddy 支持设置定时自动执行的任务，实现工作流的无人值守运行：
• 每日自动推送：每天早上定时推送 AI 热点资讯

• 每周自动汇总：每周五晚上自动生成工作周报

• 定时批量处理：每天固定时间执行数据同步、文件整理等重复性任务

配置方式：在任务创建界面中找到“定时执行”选项，设置触发时间和重复频率，WorkBuddy 就会在指定时间自动运行。

6.常见问题与避坑指南
Q1：插件/脚本安装失败怎么办？
WorkBuddy 的插件和自定义技能依赖几个核心的系统运行环境。如果安装失败，通常是环境没配齐。建议按顺序检查并安装：
• Node.js：大部分插件依赖此环境

• Python：数据分析和脚本类技能需要

• Git：部分插件安装时需从仓库拉取

装完这三个基础环境，90% 的安装报错都会自动消失。
Q2：执行任务时提示权限不足？
WorkBuddy 默认以沙箱模式运行，操作需要用户明确授权：
• 首次访问本地文件夹时会弹窗请求授权

• 高危指令（如系统级删除操作）会被拦截并要求确认

• 建议在“个人中心 → 安全设置”中检查授权范围，仅开放必要目录

Q3：网络不稳定或 API 调用失败？
• 检查网络连接，确保能正常访问 AI 模型服务

• 如果使用自定义 API，确认 Endpoint 地址和 API Key 填写无误

• 尝试切换不同的内置模型，有时某个模型可能暂时不可用

Q4：如何管理 Credits 用量？
WorkBuddy 与 CodeBuddy 共享同一账号体系和 Credits 资源：
• 可在“个人中心”查看剩余 Credits

• 每日登录签到可获取免费额度

• 新用户上线期间通常有无门槛福利（如 5000 Credits），留意官方公告
