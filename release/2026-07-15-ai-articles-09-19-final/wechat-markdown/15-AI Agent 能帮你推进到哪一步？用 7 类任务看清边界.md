# AI Agent 能帮你推进到哪一步？用 7 类任务看清边界

> **核心观点：**把“能不能做”拆成可复现的任务，才看得到 Agent 的强项和红线。

## 先说明：这不是“万能 Agent 测评”

> 本文的 7 类任务是可复现实验设计；不同模型、权限、工具连接和输入资料会产生不同结果，不能把它当作所有 Agent 的统一成绩单。

不同模型、权限、工具连接和资料质量，都会让结果不同。与其给一句“它能替代什么”的结论，不如用一套可复现的任务卡，测试它在哪些环节可靠、在哪些环节必须有人盯住。

这 7 类任务不是为了比谁更炫，而是为了问一个务实问题：**Agent 推进到哪一步时，值得你接手？**

## 任务 1：竞品调研

- **适合交给 Agent：**列出候选、收集公开资料、整理对比维度；
- **必须人工判断：**竞品定义、信息时效、结论取舍；
- **验收物：**可点击来源 + 不同观点 + 待核验项。

## 任务 2：会议纪要

- **适合：**转写、归类决定、提取待办；
- **边界：**不要把推测写成会议共识；
- **验收物：**谁在何时做什么，未决问题单列。

## 任务 3：数据分析

- **适合：**清洗思路、计算草案、图表建议、异常提示；
- **边界：**口径、缺失值、因果解释不能交给它拍板；
- **验收物：**数据字典、计算过程、可复算结果。

## 任务 4：写方案

- **适合：**搭结构、列风险、整理素材；
- **边界：**客户承诺、报价、时间表必须人工确认；
- **验收物：**目标、范围、假设和不做事项齐全。

## 任务 5：建网页或小工具

- **适合：**原型、代码、测试建议、部署说明；
- **边界：**真实密钥、支付、隐私数据、生产发布；
- **验收物：**可运行演示 + 明确测试步骤。

## 任务 6：收集资料

- **适合：**初步搜索、摘要、分类、比较；
- **边界：**来源可信度和引用原意；
- **验收物：**原始链接、发布日期、支持的具体说法。

## 任务 7：自动化流程

- **适合：**重复同步、提醒、格式转换、状态更新；
- **边界：**不可逆操作、跨系统写入、敏感数据外发；
- **验收物：**触发条件、失败告警、人工审批点、回滚办法。

## 测试时只看 4 个指标

| 指标 | 要问的问题 |
| --- | --- |
| 正确性 | 它交付的结果能否独立验证？ |
| 可控性 | 我能否看到它做了什么、停在哪里？ |
| 成本 | 节省的时间是否抵得过复核成本？ |
| 风险 | 一旦出错，是否可逆、可追责？ |

## 可采用的结论

Agent 最适合承担“有清晰输入、有明确步骤、可被验证”的推进工作。它不是一个不需要管理的虚拟员工。你越能把任务写清、把权限收窄、把验收点放在前面，Agent 越像助力；反过来，它越像一个速度很快的误差放大器。

如果你也要测试，先选一个低风险、可回滚、每周都会发生的任务，跑 7 次后再决定是否扩大权限。
