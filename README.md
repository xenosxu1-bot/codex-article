## 推荐阅读顺序
新手建议按 `01 -> 03` 阅读，先跑通界面和任务，再掌握修 bug、安全检查和常用提示词。
团队使用建议按 `02 -> 04` 阅读，重点把权限、复用能力、`AGENTS.md` 和验证标准固化下来。
扩展能力建议读 `05`，理解插件、Skill 和 MCP 的区别，再按自己的工作流选择安装。
想看完整项目开发案例，可以读 `06`，跟着 weStudy 学一遍 AI 小程序从需求到提审的闭环。
想把 AI 用进研究、写作、汇报或日常知识工作，可以读 `08`，按“完成定义—研究—生产—校验”搭建可验收工作流。
需要按功能查阅 Codex 的内置子智能体、当前会话角色模板、Skills、插件和自动化边界时，可以读 `09`。

10—19 篇组成 AI 工作方法系列，建议按需求阅读：

- 要把日常任务做成稳定交付，读 `10`、`13`、`16`、`17`。
- 要做研究、内容或小工具，读 `11`、`12`、`14`、`15`。
- 要搭一人公司协作与安全边界，读 `18`、`19`。

## 使用建议
发布到公众号时，可将 Markdown 内容复制到编辑器，再按图片引用位置插入对应图片。第 6 篇使用官方工具图片与 Mermaid 流程图，发布到不支持 Mermaid 的平台时，可先截图再插入；第 8 篇使用真实摄影，发布时请保留文末署名并复核平台的图片授权要求。
后续如果继续批量更新，可以把文章结构、配图风格、验证清单和发布前检查沉淀为一个 Codex Skill。
