# 阅读路径

> 面向第一次进入本知识库的读者。每条路径只保留完成一个具体目标所需的文章；读完后再回到[知识库总索引](<知识库总索引.md>)按主题继续查阅。

## 路径一：第一次把 AI 用成可交付的助手

**适合谁：** 会和 AI 聊天，但还没有稳定做事方法的读者。

**读完能得到：** 一套从任务描述、上下文输入到结果验收的最小闭环。

1. [Codex 新手上手：从界面到第一个高质量任务](<../01-工具教程/01-Codex 新手上手：从界面到第一个高质量任务.md>)：先学会把任务说清楚。
2. [别再只会问 AI：把它变成可交付工作流](<../03-好文方法/08-别再只会问 AI：把它变成可交付工作流.md>)：把一次提问变成可复用流程。
3. [AI 不懂你的业务，不是它不够聪明：你还没把上下文交给它](<../03-好文方法/14-AI 不懂你的业务，不是它不够聪明：你还没把上下文交给它.md>)：补齐 AI 做不好事时最常缺失的输入。

**预计阅读：** 35—50 分钟。

## 路径二：把 Codex 用进长期工作

**适合谁：** 已经完成过一次任务，想把经验沉淀为稳定工作方式的读者。

**读完能得到：** 可复用的规则、Skill 和知识库维护方法。

1. [Codex 进阶提效：配置、权限、复用与自动化](<../01-工具教程/02-Codex 进阶提效：配置、权限、复用与自动化.md>)：建立可控的日常使用环境。
2. [AGENTS.md 深度模板：让 Codex 按团队规则工作](<../01-工具教程/04-AGENTS.md 深度模板：让 Codex 按团队规则工作.md>)：把重复要求写成项目规则。
3. [Codex 插件与 Skill 指南：把常用能力沉淀进工作流](<../01-工具教程/05-Codex 插件与 Skill 指南：把常用能力沉淀进工作流.md>)：把高频流程固化成可调用能力。
4. [Codex 不只是写代码：从 0 到 1 搭建使用帮助知识库](<../05-案例实战/16-Codex 不只是写代码：从 0 到 1 搭建使用帮助知识库.md>)：看完整的知识库项目实践。

**预计阅读：** 60—90 分钟。

## 路径三：理解 Agent，并跑通第一个工具型任务

**适合谁：** 想知道 Agent、Skill、工具调用分别解决什么问题的读者。

**读完能得到：** 不依赖术语堆砌的 Agent 基础认知，以及两个可操作的工具入口。

1. [Codex 智能体与 Skills 中文参考手册](<../01-工具教程/09-Codex 智能体与 Skills 中文参考手册.md>)：建立 Agent 与 Skill 的基本概念。
2. [OpenClaw 新手上手指南：把 AI 助手接进常用聊天软件](<../01-工具教程/15-OpenClaw 新手上手指南：把 AI 助手接进常用聊天软件.md>)：了解聊天入口、Gateway 与权限边界。
3. [Hermes Agent 新手快速上手：从安装到跑通第一个 Agent](<../01-工具教程/18-Hermes Agent 新手快速上手：从安装到跑通第一个 Agent.md>)：完成一次最小 Agent 运行练习。

**预计阅读：** 45—70 分钟。

## 路径四：用案例学习“做完并验收”

**适合谁：** 不想只看概念，更想参考完整任务过程的读者。

**读完能得到：** 从失败排查、需求拆分到成果验收的案例参照。

1. [Codex 实战案例：从失败测试到可验证修复](<../05-案例实战/03-Codex 实战案例：从失败测试到可验证修复.md>)。
2. [AI 编程 Agent 实战：从想法到可验证交付](<../05-案例实战/11-AI 编程 Agent 实战：从想法到可验证交付.md>)。
3. [跟着 weStudy 做一个 AI 学习小程序](<../05-案例实战/06-跟着 weStudy 做一个 AI 学习小程序.md>)。

**预计阅读：** 60—100 分钟。先选一篇照着做，不必一次读完。
