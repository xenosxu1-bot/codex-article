# 持续更新工作流

本文件记录当前仓库的文章生产与维护流程，方便后续继续批量更新。

## 适用场景

- 查找网上 AI 热门文章并整理为选题库。
- 按“工具 / AI 知识 / 好文方法 / 安全治理”等维度批量输出文章。
- 更新仓库 README、分类索引、发布记录并推送 GitHub。

## 使用的当前项目技能

- `article-wechat`：用于选题、资料核验、中文文章写作、发布前质检和公众号风格控制。
- `github-update-push`：用于拉取远端、检查工作区、提交和推送，禁止强推。
- `imagegen`：仅在需要 AI 生成配图时使用；必须记录 AI 视觉来源，不伪装成实拍或官方图。
- `openai-docs`：涉及 OpenAI / Codex / API / 模型选择等官方信息时优先使用官方文档核验。

## 标准流程

1. **同步仓库**：`git status -sb`、`git fetch --prune origin`，确认本地改动属于当前任务。
2. **扫描既有内容**：读取顶层 `NN-*.md`、`README.md`、`CONTENT_INDEX.md`，避免同题重复。
3. **检索热门资料**：优先官方公告、研究报告、权威媒体和论文周榜；二手媒体只作为线索。
4. **分类与打分**：按工具、AI 知识、好文方法、安全治理、企业落地等维度归类。
5. **写作**：每篇文章必须包含清楚结论、读者场景、可执行动作和“资料来源与边界”。
6. **本地检查**：检查 Markdown 代码块、图片引用、相对链接、乱码、标题重复和来源链接。
7. **维护编排**：更新 `README.md`、`CONTENT_INDEX.md` 和 `publish-record.md`。
8. **提交推送**：只暂存本次相关文件，提交前再次 `git diff --cached --stat`，推送 `main`，禁止强推。

## 文章完成标准

一篇文章进入仓库前，至少满足：

- 标题与正文一致，不制造恐慌或夸大承诺。
- 关键事实有来源；涉及最新内容写明访问日期。
- 事实、推断、观点分开表达。
- 不虚构客户案例、亲身测试、效率数据或采访。
- 对高风险内容给出边界，不提供医疗、法律、金融等专业建议。
- Markdown 结构完整，代码块闭合，相对链接可解析。

## 批量更新建议

- 每批建议 5—10 篇，避免主题跨度过大。
- 每批至少包含 1 篇安全或治理文章，平衡效率叙事。
- 热门追踪文章建议保留资料来源小节，方便后续复核。
- 每次批量完成后，用 `CONTENT_INDEX.md` 重新编排系列，而不是只追加到 README 末尾。

## 可沉淀为 Skill 的部分

如果后续持续做这类任务，可以创建一个“AI 热门文章雷达”Skill，记录：

- 固定检索源；
- 热度判断标准；
- 选题评分卡；
- 文章模板；
- 来源台账格式；
- README / CONTENT_INDEX 自动更新规则。

如果要每周自动扫描和提醒，则更适合做 Automation，并接入稳定搜索源或 RSS/MCP 数据源。
