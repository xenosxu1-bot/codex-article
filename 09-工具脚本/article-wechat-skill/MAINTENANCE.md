# article-wechat Skill 维护规则

## 修改边界

- 修改 `SKILL.md`：仅用于触发条件、任务模式、启动顺序、不可违反的底线和默认行为。
- 修改 `references/`：用于详细流程、写作规范、事实核查、视觉版权、发布规则和质检清单。
- 修改 `references/templates/`：用于标准产物模板。
- 修改 `references/recurring-rules.md`：只记录长期有效、可复用、会影响质量或安全的规则。

## 不应写入 Skill 的内容

- 账号密码、密钥、Cookie、OAuth token。
- 未公开商业资料、读者隐私、客户隐私。
- 单篇文章的一次性偏好。
- 未验证的流量结论或个人猜测。

## 自检清单

- [ ] `SKILL.md` 保持入口化，不塞入长篇细则。
- [ ] 发布规则和交付路径没有冲突。
- [ ] 新增 reference 文件已被 `SKILL.md` 或相关文档引用。
- [ ] `recurring-rules.md` 中冲突旧规则已标记为 `replaced`。
- [ ] 默认不发布，只有用户明确要求才发布。
- [ ] 模板文件可以直接复制使用。
- [ ] 没有写入账号密钥、Cookie、隐私或未公开资料。

## 建议迭代节奏

- 每次重要修改更新 `CHANGELOG.md`。
- 每 5-10 篇文章复盘一次长期规则，清理过期或冲突规则。
- 接入新仓库、公众号后台或数据系统时，优先新增独立发布/集成文档，不把账号配置混入通用写作规则。
