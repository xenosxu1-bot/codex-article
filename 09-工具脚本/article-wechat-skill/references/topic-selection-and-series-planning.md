# AI 自媒体选题筛选与系列编排流程

> 用于“热门 AI 工具使用”“AI 学习教程”“内容规划”“栏目策划”“连续发文”类任务。目标不是追逐工具名，而是稳定产出有证据、可执行、可系列化且不与既有内容重复的文章。

## 一、先确定任务模式

| 用户请求 | 先做什么 | 默认输出 | 是否写文 |
| --- | --- | --- | --- |
| “给我热门选题” | 选题去重、趋势判断、评分 | 10—20 张选题卡 + 优先级 | 否 |
| “规划一个系列” | 栏目定位、读者分层、选题地图 | 系列主线、篇目、顺序、复用素材 | 否 |
| “做前五篇/按顺序写” | 逐篇 Brief、资料、图文、质检 | 完整生产包 | 是 |
| “发布/上传/推送” | 重核验并按编号发布 | GitHub 发布记录 | 是，且发布 |

只有出现“发布 / 上传 / 推送 / 同步到 GitHub”时，才进入发布流程。

## 二、选题前的去重与边界检查

1. 读取 recurring-rules.md。
2. 扫描本地 articles 目录下的 README、Brief 和标题，记录已覆盖的主题、角度与承诺。
3. 若任务涉及发布，再额外读取目标仓库顶层文章与 README；不因选题阶段而修改仓库。
4. 把候选拆成：
   - 工具使用：连接器、数据、图文视频、办公、开发；
   - 工作方法：上下文、任务卡、验收、归档、协作；
   - 安全治理：权限、数据、版权、提示注入、审校；
   - 学习教程：能力地图、练习、反馈、复盘、Skill；
   - 行业/岗位：内容、运营、销售、管理、个体创业。
5. 对已写主题，必须寻找新读者、新问题、新证据或新交付物；只有换标题、不换价值，不进入优先队列。

## 三、如何判断“热门”而不追逐噱头

“热门”至少满足以下一项：

- 主流产品或标准出现实质方向变化，且能找到一手资料；
- 目标读者的高频工作任务发生变化；
- 新能力带来新的错误、权限、版权或验证问题；
- 可拆为多篇互相引用的教程系列。

### 时效性规则

- 涉及“最新、刚发布、价格、功能、模型、平台规则、市场数据”时，必须先查一手资料并写明截至日期。
- 没有可信热度数据时，使用“值得关注的方向”“近期产品方向”“高频使用场景”，不要伪称“全网最热”“爆款保证”。
- 不用单一社交平台热搜、二手测评或虚构个人体验支撑关键结论。

## 四、选题评分卡（总分 25，风险单独扣分）

### 100 分制优先级换算

需要全自动排序或批量生产时，可换算为 100 分：读者刚需 25、差异化角度 20、证据可得性 20、可执行交付物 20、系列延展性 10、制作成本 5。无法核验关键事实扣 30，只有工具新闻没有方法沉淀扣 20，与既有文章高度重复扣 20，标题承诺过大扣 15，高风险建议缺少边界扣 30。低于 75 分不进入完整生产包。


| 维度 | 0 分 | 3 分 | 5 分 |
| --- | --- | --- | --- |
| 读者需求 | 纯工具新闻 | 有明确场景 | 高频、刚需且能立即行动 |
| 差异化 | 与旧文同质 | 换了读者或例子 | 有独特交付物、框架或证据 |
| 证据可得性 | 只能凭感觉 | 有少量可信资料 | 有一手资料与可核验演示边界 |
| 系列延展性 | 单篇即结束 | 可关联 1—2 篇 | 能形成 4—6 篇内容主线 |
| 生产可行性 | 需要未公开资料 | 需较多验证 | 可在现有资料、时间与版权边界内完成 |

风险扣分：医疗、法律、金融、高敏数据、侵权风险、无法核验的数字/案例、过度依赖短期产品功能，各扣 1—5 分。低于 15 分或证据不可用的候选不建议优先写。

## 五、标准选题卡

每个候选至少包含：

~~~text
标题：
目标读者：
核心承诺：读完能拿走什么？
读者真正卡住的问题：
核心观点（1 句）：
独特角度：与已写文章的区别是什么？
可交付物：清单 / 模板 / 步骤 / 对照表 / 提示词 / 练习
证据与资料：需要哪些一手来源？
图片方向：真实摄影、官方截图或标注的 AI 视觉？
适合平台：公众号 / 知乎 / 小红书 / 掘金等
评分：需求 / 差异化 / 证据 / 系列 / 可行性 / 风险
推荐顺序：P0 / P1 / P2
~~~

## 六、系列编排原则

一条高质量 AI 自媒体系列，宜按“认知 → 操作 → 验收 → 边界 → 复用”编排：

~~~text
认知：它解决什么 → 操作：怎样开始 → 验收：怎么判断结果 → 边界：权限、数据、版权 → 复用：模板、Skill、工作台
~~~

### 2026-07-15 基准选题顺序（可按新证据重排）

1. **MCP 到底是什么？不用懂代码，也能看懂 AI 为什么开始“会用工具”了**：认知 + 低风险连接入门。
2. **AI 不懂你的业务，不是它不够聪明：你还没把“上下文”交给它**：项目资料、规则和决策的组织方法。
3. **2026 年学 AI，不要从 100 个 Prompt 开始：一份 30 天能力地图**：从工具焦虑转向能力训练。
4. **AI 做 PPT，最容易翻车的不是排版，而是这 4 个“内容失真点”**：办公场景的验收方法。
5. **AI 能操作电脑了，普通人先别急着全自动：一份权限与验收清单**：执行型 Agent 的安全边界。

这五篇组成“工具连接 → 上下文 → 学习 → 办公输出 → 执行边界”的连续阅读路径。后续可续写：AI 表格分析、个人知识库、视频生产、提示词评分表、Agent 任务审计。

## 七、从选题到生产包的交接清单

一个候选进入写作前，必须确认：

- [ ] 标题不是既有文章的换皮版本。
- [ ] 正文前 30% 能回应标题承诺。
- [ ] 至少有可交付的清单、模板、步骤或验收表。
- [ ] 有足够的一手资料支撑时效性表述；否则改为方法论文章。
- [ ] 明确图像类型与授权来源；真实图片要本地保存并记台账。
- [ ] 不使用虚构的第一人称测试、客户案例、效率数字或访谈。
- [ ] 若标题含“我用 / 我测试”，正文必须给出可复现的测试条件或改成非第一人称标题。

## 八、复盘后如何更新选题库

文章完成后，在同一生产包的 Brief 或复盘中记录：

- 它解决了哪个读者问题；
- 哪个框架、清单或视觉主题可以复用；
- 哪条事实或功能最容易过期；
- 下一篇最自然的衔接是什么；
- 是否发现新的长期规则（仅在符合 recurring-rules 的升级标准时写入）。

不要把账号、Cookie、访问令牌、未公开数据或单篇偶然的流量结论写进 Skill。
