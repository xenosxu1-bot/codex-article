# 全自动文章生产工作流

用于“完整生产包”“按方案写文章”“批量生产文章”“从选题到图文包”任务。目标是从主题输入到可发布生产包自动完成；遇到发布授权、高风险内容、无法核验事实或 P0/P1 问题时自动阻断或标记待确认，不强行发布。

## 默认流水线

1. **输入解析**：确认主题、目标读者、文章类型、目标动作、资料要求、图片要求和发布要求；缺失时采用 Skill 默认假设并在交付说明标记。
2. **选题去重与评分**：扫描既有文章、Brief、标题和系列；生成候选角度并按读者需求、差异化、证据、交付物、系列延展和生产可行性评分。
3. **Brief**：生成内容 Brief，锁定核心承诺、事实边界、可复用资产、图片计划和入库位置。
4. **资料与来源台账**：涉及时效、产品、模型、价格、政策、数据、平台规则或高风险建议时，优先核验一手来源并记录访问日期。
5. **标题候选与评分**：生成 8-12 个标题，准确性、具体性、读者收益、辨识度低于阈值不得作为主标题。
6. **结构地图与插图分镜**：先设计标题承诺、首屏回应、H2 承接、正文插图位置和结尾回收，再写正文。
7. **初稿生成**：按文章类型骨架写正文；不写 H1、frontmatter、参考资料区块、发布前检查或 Markdown 分割线。
8. **关键断言核查**：抽取事实、数据、产品能力、价格、政策、推断、观点和经验，逐条匹配来源或降级表达。
9. **去 AI 化改写**：删除宏大开头、空泛词、无证据效率数字和口号式结尾；增加具体场景、类比、动作、例子和验收标准。
10. **视觉生成**：生成封面 Brief、无文字底图、本地中文叠字和正文插图；写入图片台账与生成记录。
11. **格式与移动端检查**：检查标题层级、段落长度、图片路径、加粗、表格、代码块和微信兼容性。
12. **独立 QA**：按 P0/P1/P2 输出问题清单；P0/P1 未清零不得标记为可发布。
13. **生产包归档**：输出正文、Brief、标题评分、来源台账、断言台账、结构地图、插图分镜、图片台账、去 AI 化报告、格式报告、QA 报告和发布记录。
14. **可选发布**：只有用户明确说发布、上传、推送或同步到 GitHub 时，才进入 publishing-github.md。

## 自动通过阈值

- 选题总分建议不低于 75/100，且必须有明确可交付物。
- 关键事实、产品能力、价格、政策、平台规则和数据必须有来源；无来源则删除、降级或待确认。
- 标题承诺必须在正文前 30% 得到回应。
- 深度稿至少包含 1 个可复用资产和 2 张正文图。
- P0/P1 问题必须清零；P2 可记录为后续优化。

## 阻断条件

出现以下任一情况，停止“可发布”交付，只输出修复清单或待确认项：

- 关键事实无来源或来源与结论不匹配。
- 虚构测试、案例、采访、客户反馈、效率倍数或第一人称经历。
- 图片授权不清，或把 AI 视觉伪装为实拍、官方截图、真实新闻或真实客户案例。
- 涉及医疗、法律、金融、高敏数据、隐私、平台规则但没有清楚边界。
- 标题、封面、摘要与正文承诺明显不一致。

## 外部能力建议

- 实时资料、价格、法规、产品功能：使用浏览器、搜索 API 或研究 MCP，并保留来源台账。
- 图片生成、OCR、视觉检查：使用图像生成 Skill、视觉检查工具或本地脚本。
- 公众号草稿、发布和数据复盘：使用带 OAuth 的 MCP、插件或官方接口；不要把 Cookie、密钥或账号信息写入 Skill 或仓库。
