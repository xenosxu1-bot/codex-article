用于完整生产包、深度稿、教程、Agent 实战、流程拆解、方法清单和知识库文章。目标是让每张图服务正文，而不是作为装饰。

## 插图选择

| 文章内容 | 推荐插图 |
| --- | --- |
| 工具教程 | 步骤流程图、界面区域说明图 |
| 方法论文章 | 方法路径图、任务卡片图 |
| 观点分析 | 对比矩阵、判断框架图 |
| Agent 实战 | 执行链路图、权限边界图 |
| 选题规划 | 系列地图、优先级矩阵 |
| 风险治理 | 风险分层图、检查清单图 |

## 生成原则

- 深度稿至少 2 张正文图：1 张流程/架构图，1 张检查卡/任务卡/验收图。
- 优先使用确定性图表：Mermaid、SVG、Python 绘图或本地排版卡片；需要氛围或概念表达时再用 AI 视觉。
- Mermaid 必须转 PNG 后插入正文；微信公众号发布版不保留 Mermaid 代码块。
- 官方截图必须来自官方页面或用户提供素材，并记录来源和使用范围。
- 自制或重绘 UI 必须标注“自制/重绘示意”，不能冒充官方界面。
- AI 视觉必须在图片台账或生成记录中标注，不能作为真实证据。

## 正文承接规则

正文中只保留可渲染 Markdown 图片语法和自然承接句，不保留素材说明堆叠。

推荐结构：

~~~md
这套流程可以拆成四个动作：先定义任务，再整理资料，接着让 AI 生成中间稿，最后用人工验收标准收口。

正文插图路径示例：`../08-素材库/图片/正文插图/23-AI任务执行流程-正文插图01.png`

真正决定质量的不是模型名字，而是最后一步有没有验收标准。
~~~

## 图片检查

- 图片文件存在，路径可渲染。
- 含中文或空格的本地图片路径使用尖括号包裹。
- Alt 文本完整且能说明图片功能。
- 图片中文字清晰可读，重要信息在正文中也有文字承接。
- 图片已写入 image-ledger.md、图片生成记录或文章资产登记表。
- 没有乱码、真实 Logo 冒用、水印、二维码、未授权人物肖像或真实 UI 伪造。
