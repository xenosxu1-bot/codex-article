# GitHub 编号发布规则

## 触发条件

只有用户明确要求“发布”“上传”“推送”“同步到 GitHub”或等价表达时，才执行本流程。写草稿、做完整生产包、审核修订均不自动发布。

## 默认目标

- 仓库：`https://github.com/xenosxu1-bot/codex-article`
- 分支：`main`
- 安全规则：不得强推；不得覆盖已发布文章；先获取远端更新。

## 发布步骤

1. 获取远端最新状态，确认工作副本干净或处于已知状态。
2. 读取知识库索引和各中文分类目录中的 `NN-*.md` 正式文章，使用下一个未占用的两位数编号；已发布文章不得重编号。
3. 使用同一编号命名文章和新增图片；文章放入对应中文分类目录，图片放入素材库目录。
4. 更新根目录 `README.md`、知识库首页、总索引、分类索引、系列索引、标签索引和必要的分类目录 README，并按编号升序排列。
5. 完成事实、版权、格式、链接和图片路径检查。
6. 暂存、提交并推送到 `main`；不得强推。
7. 在 `07-资料与流程/发布记录.md` 或项目约定的发布记录文件中记录提交哈希、远端链接和待观察项。

## 发布前必须确认

- 目标仓库和分支正确。
- 本次新增/修改文件清单清楚。
- 图片来源与授权已记录。
- 标题、封面、正文一致。
- 没有账号密钥、Cookie、隐私数据或未公开资料。
- 总索引、分类索引、系列索引、标签索引和发布记录已同步更新。

## 知识库仓库约定

若目标仓库采用中文知识库结构，发布时同时遵循 `references/article-knowledge-base-workflow.md`：

- 正式文章不放在根目录，按主分类放入中文目录。
- 根目录通常只保留 `README.md`、`AGENTS.md` 等平台约定文件。
- 图片统一放入素材库，例如 `08-素材库/图片/`。
- 发布包和脚本不计入正式文章总数。
- 每次发布或重排后同步更新索引和发布记录。

## 失败处理

- 推送失败时不重试破坏性操作；先报告原因和当前本地状态。
- 发生冲突时先同步远端并保留双方改动，不静默覆盖。
- 不创建后台定时任务；如需定期发布或复盘，建议另设 Automation。
