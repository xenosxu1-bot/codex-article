# AI 自媒体生产系统参考

本参考用于“完整生产包”、账号长期运营、团队协作或需要搭建智能体流程时加载。目标不是让 AI 无人值守地批量发文，而是让人负责定位、判断和签字，让 AI 提高信息处理、表达、检查和复盘效率。

## 一、系统目标与控制原则

- **目标：** 稳定产出有明确读者价值、可核验、视觉一致、可复盘的内容，而非单纯提高发文数量。
- **人负责：** 账号定位、价值判断、敏感议题、关键事实确认、版权确认、最终发布。
- **AI 负责：** 选题发散、资料整理、结构草拟、标题变体、视觉 Brief、格式检查、问题清单和复盘归纳。
- **四道门：** 选题门、事实门、视觉/版权门、发布门。任一门未通过，文章保持“待审”，不能自动发布。

## 二、端到端流程

| 阶段 | 关键问题 | 最小产物 | 通过标准 |
| --- | --- | --- | --- |
| 1. 选题池 | 写给谁，解决什么问题，为什么现在写 | 选题卡与评分 | 有目标读者、真实需求、差异化与证据路径 |
| 2. 内容 Brief | 文章要让读者相信或完成什么 | 一页 Brief | 一句主张、边界、结构、CTA 明确 |
| 3. 研究 | 哪些事实必须核验，哪些只是观点 | 来源台账 | 关键说法可追溯，未知项明确标记 |
| 4. 正文 | 怎样让读者顺畅读完并采取行动 | Markdown 初稿 | 场景、论证、方法和结尾闭环 |
| 5. 标题/封面 | 怎样准确表达价值并获得点击 | 标题组、视觉 Brief、素材清单 | 不夸张，和正文一致，识别度足够 |
| 6. 配图/格式 | 如何适配移动端并降低理解成本 | 本地图片、排版稿 | 图片相关、版权清楚、结构易扫读 |
| 7. 质检 | 哪些错误会伤害信任或造成风险 | AI 质检报告 | 高风险问题全部关闭或显式保留待审 |
| 8. 发布/复盘 | 是否可发布，下一次如何更好 | 发布记录、指标复盘 | 发布动作可追溯，有下一次假设 |

## 三、智能体分工与交接

规模小（单人、每周 1–3 篇）时，一个主智能体按顺序切换角色即可；规模大或多人协作时，再拆分为独立智能体。每个角色只接收上一步产物，不直接越过验收门。

| 智能体 | 输入 | 输出 | 不可替代的人类判断 |
| --- | --- | --- | --- |
| 选题策划 | 账号定位、读者、内容数据、热点线索 | 选题池、评分、切入角度 | 是否符合长期品牌与商业目标 |
| 研究核验 | 内容 Brief、关键说法 | 来源台账、事实/观点边界、风险提示 | 一手来源选择与关键结论确认 |
| 结构与写作 | 已确认资料、主张、语气 | 结构、初稿、提示词/案例 | 作者立场、真实经历与最终表达 |
| 标题编辑 | 正文、读者、渠道 | 8–12 个标题、主副标题、选择依据 | 是否符合品牌承诺与读者预期 |
| 视觉策划 | 文章主张、视觉规范、可用素材 | 封面 Brief、配图清单、图注与署名 | 图片真实性、授权与品牌审美 |
| 格式编辑 | 正文、图片、发布渠道 | 移动端排版稿、链接与素材检查 | 最终呈现与发布渠道规范 |
| AI 质检 | 成品、来源台账、素材清单 | 按风险级别排序的问题清单 | 是否接受风险、是否允许发布 |
| 复盘分析 | 发布数据、评论、转化线索 | 可验证洞察、下一轮实验假设 | 指标优先级与试验取舍 |

### 推荐交接字段

每个产物至少包含：`文章编号`、`版本`、`负责人/智能体`、`状态（草稿/待审/已通过/已发布）`、`关键假设`、`待确认项`、`最后更新时间`。这样可以避免不同智能体把推断当事实。

## 四、选题系统

### 选题卡模板

```text
主题：
目标读者：
读者正在经历的问题：
文章承诺的具体收益：
独特角度：
必须核验的事实：
可引用的一手来源：
风险与不写的边界：
行动号召：
评分（相关性 / 需求 / 证据 / 差异化 / 可传播性）：
```

### 建议的内容配比

- **常青内容：** 方法、清单、框架、工具使用边界，积累搜索与收藏价值。
- **趋势解读：** 新工具、新政策、新现象；只在来源充分时发布，注明具体日期。
- **案例拆解：** 用可验证事实讲过程、限制和复盘，不用“神话式成功故事”。
- **观点内容：** 用明确立场吸引讨论，同时写出事实依据与反对意见。
- **转化内容：** 明确服务、产品或行动路径，避免将广告伪装成客观评测。

不要机械追求固定比例；以账号定位、读者反馈和资源能力为准。每周维护选题池，优先写“高需求 + 高证据 + 高差异化”的交集。

## 五、标题、封面与配图规范

### 标题

1. 为同一篇文章生成 8–12 个候选：利益型、问题型、反常识型、案例型至少各 1 个。
2. 用 100 分评分：准确性 30、具体性 25、读者收益 25、辨识度 20。低于 75 分不建议发布。
3. 主标题承诺的价值必须在正文前 30% 得到回应；标题中的数字、对象、结果与正文严格一致。
4. 禁止假稀缺、未证实对比、恐吓式措辞、无依据的“最强/必看/彻底改变”。

### 封面

封面 Brief 必须写清：核心画面、主文案（如有）、阅读情绪、主要颜色、主体位置、裁切安全区、禁止元素和署名/素材来源。封面只表达一个主题；缩小后仍能识别主题；不要把关键事实塞进图片。

### 配图

- 真实摄影：记录来源页、作者/机构、许可、下载日期、原图链接和本地文件名。
- 官方素材：记录官方页面与使用范围，不擅自修改商标或伪造产品界面。
- AI 视觉：在图注和来源表中明确标注“AI 视觉”；不作为真实新闻、真实客户案例或实拍证明。
- 每张图对应一个阅读功能：建立氛围、解释流程、展示证据、分隔节奏或帮助记忆。没有功能就删除。

## 六、格式与移动端体验

- 标题后尽快进入场景或结论，避免长前言。
- 普通段落以 1–3 个句子为主；长段落拆分为小标题、列表或引用。
- 一屏只承载一个主要信息；重要结论用短句、加粗或引用提示，不依赖颜色区分。
- 表格只用于横向比较；移动端难读时改成分点清单。
- 图片前后写清它回答什么问题；图片内文字要有足够对比，且正文中保留同等信息。
- 发布前在目标编辑器预览；不同平台的封面裁切和图片压缩规则以当日后台规范为准。

## 七、AI 检查智能体

### 检查提示词

```text
你是独立的中文内容审稿与风险检查智能体。不要润色，不要改写正文；只输出问题清单。

请按 P0（不能发布）、P1（发布前必须修）、P2（建议优化）分类检查：
1. 事实、数据、日期、产品能力与来源是否可核验；
2. 事实、推断、观点和假设是否混淆；
3. 标题、封面、摘要和正文是否存在过度承诺或不一致；
4. 是否有空泛、重复、跳跃、读者错位、营销伪装或敏感风险；
5. 图片是否真实匹配、文件是否存在、署名/授权/AI 生成标注是否完整；
6. Markdown 标题、链接、脚注、表格、代码块和移动端阅读是否有问题；
7. 结尾是否给出合适且不强迫的下一步。

每项问题必须包含：位置、风险原因、修复建议、是否需要人工确认。
```

### 发布阻断项

出现下列任一情形，状态必须为“待审”：关键事实无来源、引用与结论不匹配、图片授权不清、把 AI 图伪装为实拍、标题与正文严重不符、涉及医疗/法律/金融等高风险建议但未标明边界，或存在明显隐私/诽谤风险。

## 八、数据复盘与知识沉淀

建立内容台账；每篇至少记录：文章编号、主题、目标读者、发布时间、标题版本、封面版本、首段钩子、CTA、阅读/收藏/分享/评论/转化数据、典型反馈、结论可信度和下一次假设。

复盘问题：

1. 哪个读者问题被真正解决，而不是哪个标题“看起来更炸”？
2. 流失发生在标题、首屏、结构、案例、视觉还是 CTA？
3. 哪项结果来自可重复因素，哪项只是热点、分发或样本波动？
4. 下次只改变一个关键变量，如何设计可比较的试验？

将经过多次验证的规则写入 `recurring-rules.md`；单篇偶然数据只保留为假设，不升级为硬规则。

## 九、工具与集成建议

| 能力 | 推荐接入方式 | 目的 |
| --- | --- | --- |
| 一手资料研究 | 浏览器、搜索 API 或研究 MCP | 保存来源、日期与证据片段 |
| 素材管理 | 图片库、对象存储或受控本地目录 | 记录许可、来源和本地文件名 |
| AI 视觉 | 图像生成 Skill / 工具 | 生成封面概念图，并明确标注 AI 视觉 |
| 文稿与质量检查 | 本 Skill + 独立审稿角色 | 输出结构化文章和可追溯检查报告 |
| 仓库发布 | `publishing-github.md` 中的内置发布流程 | 编号、版本控制、审核与可回滚提交 |
| 公众号草稿/数据 | 经 OAuth 授权的 MCP、插件或官方接口 | 减少手工复制，保留发布与数据记录 |

不要把账号登录信息、密钥、未公开资料或读者隐私写入 Skill、提示词或 Git 仓库。接入任何发布或数据系统前，先明确权限范围、审批人、失败回滚和审计记录。

## 十、落地路线

- **第 1 阶段（MVP）：** 一个账号、一种内容类型、一份选题表、一个 Brief、人工发布；先跑通每周稳定产出与质检。
- **第 2 阶段（标准化）：** 固定标题评分、视觉 Brief、来源台账、质量报告和 GitHub 归档；开始记录发布数据。
- **第 3 阶段（半自动化）：** 通过 MCP/插件连接研究、素材、草稿和数据系统；发布仍保留人工审批门。
- **第 4 阶段（实验驱动）：** 以主题、钩子、标题、封面和 CTA 为变量进行小规模对照，持续更新长期规则。
