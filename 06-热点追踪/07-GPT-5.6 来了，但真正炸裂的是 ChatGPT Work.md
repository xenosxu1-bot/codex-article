> OpenAI 7月9日（北京时间7月10日凌晨）搞了个大的，不是挤牙膏，是一口气倒出来三件事：GPT-5.6 全量开放、ChatGPT Work 智能体上线、Codex 直接塞进 ChatGPT 桌面端。这篇文章不打算面面俱到抄一遍新闻稿，我想聊点不一样的。

## 一、先说 GPT-5.6：名字变了，思路也变了

GPT-5.6 不是一个模型，是三个：Sol、Terra、Luna。

以前 OpenAI 发新模型很简单——出一个更强的，旧的降价。但这次他们搞了一套分层定价，三个模型各有明确定位，价格差 5 倍。

Sol 在编码测试 Terminal-Bench 2.1 上拿了 91.9%，确实猛。但我猜大部分普通用户根本用不上 Sol——它的定价是 Luna 的 5 倍，除非你在做网络安全分析或者复杂的科研推理，否则 Terra 才是那个甜点区。

有意思的是什么呢？OpenAI 这次把网络安全作为 GPT-5.6 的核心卖点来推。在特朗普政府时期，他们曾经被限制过这方面的推广，现在放开了手脚。这个转向其实挺耐人寻味的——说明 OpenAI 在 B 端的安全合规市场上，有很明确的野心。

### 说说我的实测感受：

Luna 的反应速度真的快，体感上和 GPT-4o mini 差不多，但逻辑能力明显上了一个台阶。日常写作、翻译、聊天，Luna 完全够用，没必要开更贵的模型。

Terra 是那个万金油。写代码、分析文档、做一些中等复杂度的推理，它都能干，而且价格合理。我猜大部分 Plus 用户默认跑的就是这个。

Sol 嘛……打开它之前，先想清楚自己是不是真的需要。它的深度推理 Max 模式和多智能体协同 Ultra 模式听起来很唬人，但绝大多数场景用不上。

## 二、ChatGPT Work：Agent Mode 的第三次改名，这次来真的了

如果你一直在关注 OpenAI，应该记得他们给智能体功能改过好几次名：

- 最早叫 Agent Mode
- 后来改过什么名字我都不太记得了
- 现在是 ChatGPT Work

但这一次不只是改名字。

之前的 Agent Mode 口碑并不好——跑个几分钟就断，稍微复杂一点的任务就卡住，用户吐槽还不如我自己干。ChatGPT Work 的核心改进就一条：能连续干活数小时了。

具体能干吗？

- 跨应用操作：它可以在你的桌面应用、浏览器、代码编辑器之间来回切换
- 长周期任务：比如帮我整理这周的竞品分析报告，从这 20 个网页里提取信息，做成 PPT——以前你得一步步喂，现在扔给它就行
- 内置 Codex：这意味着它能直接写代码、跑测试、提 PR

说实话，ChatGPT Work 才是这次更新里最值得关注的东西。GPT-5.6 再好也只是模型能力的提升，但 ChatGPT Work 改变的是人跟 AI 的交互方式——从一个对话窗口，变成一个能帮你干活的同事。

不过话说回来，智能体这个概念喊了两年了，每次发布都说这次真的不一样。好不好用，还得等落地之后看实测。

## 三、Codex 并入 ChatGPT：打工人桌面要变天了

这是一个很多人忽略但实际上影响很大的变化。

之前 Codex 是一个独立的桌面应用，专门给开发者用的。这次 OpenAI 直接把它合并进了 ChatGPT 桌面端。更新后的 ChatGPT 桌面应用有三个模块：Chat（聊天）、ChatGPT Work（工作）、Codex（编程）。

这意味着什么？

以前是两条线：
- 开发者用 Codex → 写代码
- 普通用户用 ChatGPT → 聊天

现在是一条线：
- 所有人都用同一个桌面应用，想聊天聊天，想写代码写代码，想干活干活

Codex 并入 ChatGPT 这个动作，让我想到微软把 Copilot 嵌进 Office——不是功能合并，是生态整合。OpenAI 在告诉市场：我不只是做聊天机器人的，我是做AI 工作平台的。

对开发者来说，新版的 Codex 功能也有了实质提升：

- 支持多仓库项目管理
- PR 审查能力增强
- Diff 内联编辑
- Developer Mode
- 手机远程控制

## 四、GPT-Live：语音助手终于能边听边说了

同样是这两天的事，OpenAI 发布了 GPT-Live 全双工语音模型。

之前的语音模式是你一句我一句——你说完，它再说。GPT-Live 打破了这种排队模式，实现了真正的全双工通信：它能边听边说，你打断它，它能自然接住；它思考的时候，会用 '嗯哼' 这样的反馈音让你知道它还在。

这东西的实际体验提升有多大？用过 Apple Watch 的 Siri 再对比一下就知道——语音助手好不好用，关键不在识别率，而在交互的自然度。一个能自然打断、自然插话的语音助手，和那种每次都要等它说完才能接话的，完全是两个物种。

但是，GPT-Live 目前只在 Plus 和 Pro 套餐里可用，而且刚开始只在美国地区推送，国内用户想用还得等等。

## 五、一圈看下来，我自己的感受

1. 模型内卷到头了。GPT-5.6 很强，但和 GPT-5.5 的差距没有以前从 GPT-4 到 GPT-4o 那么大。巨头们都在从堆参数转向做产品。Sol/Terra/Luna 的分层定价就是信号。

2. 智能体才是下一个战场。ChatGPT Work 如果真能稳定跑几个小时不崩，那它的影响会比任何模型升级都大。问题是——OpenAI 的智能体之前翻过车，这次能不能兑现，我持谨慎乐观。

3. Codex 并入 ChatGPT 桌面端这个事，长期来看可能被低估了。开发者工具和生产力工具的合流，意味着 OpenAI 在抢微软的盘子——毕竟微软的 Copilot 也在做同样的事。

4. GPT-Live 的体验升级被低估了。语音助手的交互范式从排队变成对话，这个跨越比跑分提升几个点重要得多。

## 整合后的使用方式

这篇文章在整合时保留了原稿里最有价值的判断，同时补上了读者真正落地时容易缺失的三类信息：第一，适用边界；第二，执行顺序；第三，验收标准。阅读时不要只把它当成观点文，更适合把它当成一张操作说明书：先确认自己的场景是否匹配，再照着清单完成一次小规模试跑，最后把有效步骤沉淀成固定模板。

一个更稳的做法是从低风险任务开始。不要一上来就把账号、资金、客户数据或正式发布权限交给 AI，也不要要求它一次完成从调研到发布的全部环节。先选择一个结果可回滚、材料可替换、责任人明确的小任务，让 AI 负责收集、整理、对比、起草或检查其中一段，人负责确认目标、边界和最终判断。这样做的好处是，出错成本低，复盘线索清楚，也更容易看出到底是提示词问题、资料问题、工具权限问题，还是验收标准本身没有写清楚。

## 可以直接照抄的执行清单

开始前先写四句话：我要完成什么；结果交给谁；哪些内容不能编；什么情况必须停下来问我。执行中保留三份材料：任务卡、过程记录和最终版本。验收时至少检查五件事：事实是否可追溯，结论是否超过证据，格式是否适合目标渠道，风险是否被标出，下一次能否复用这套步骤。只要这五项没有通过，就不要把 AI 的输出当成最终稿，而应当把它看成一个需要继续加工的半成品。

## 常见误区与修正动作

最常见的误区，是把“AI 给了答案”等同于“任务已经完成”。真正的完成，必须能被另一个人接手、检查和复用。第二个误区，是在没有上下文的情况下追求更长的提示词。提示词越长不一定越可靠，关键是里面有没有目标、材料、边界、样例和验收口径。第三个误区，是只保存最后结果，不保存中间判断。这样下次遇到类似任务时，仍然要重新解释一遍背景，也无法知道上次哪里有效、哪里只是碰巧。

更好的修正动作很简单：每次只改一个变量。先固定任务卡，再换资料；先固定资料，再换输出结构；先固定结构，再换语气。这样连续做两三轮后，你得到的不只是一次回答，而是一套可以复用的个人或团队工作流。
