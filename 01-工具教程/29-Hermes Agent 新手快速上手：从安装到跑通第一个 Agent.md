> 一句话结论：Hermes Agent 新手上手，先不要急着接 Gateway、MCP、Skills 和自动化。第一天只做一件事：按 **安装 → 选模型 → 跑通对话 → 恢复会话 → 再加工具** 的顺序，把最小闭环跑稳。

如果你最近在看开源 AI Agent，大概率会被 Hermes Agent 的功能列表吸引：终端对话、Web UI、消息入口、工具集、Skills、记忆、MCP、Cron、远程运行、团队协作。

这些能力都很有想象力，但对新手来说，也很容易变成负担。你还没确认模型能不能正常回复，就去接 Telegram；还没理解工具权限，就打开执行命令；还没跑通本机 CLI，就把它部署到服务器。最后问题一多，根本分不清是模型、网络、权限、配置，还是工具调用出了问题。

这篇文章基于 [NousResearch/hermes-agent 官方仓库](https://github.com/NousResearch/hermes-agent) 和 [Hermes Agent 官方 Quickstart](https://hermes-agent.nousresearch.com/docs/getting-started/quickstart) 的公开信息整理，适合第一次接触 Hermes Agent 的读者。目标不是把所有功能讲完，而是帮你先跑起来，并知道下一步该怎么加能力。

## 先用一句话理解 Hermes Agent

Hermes Agent 可以理解成一个开源的 AI Agent 工作台。它不是只有一个聊天窗口，而是把模型、工具、会话、记忆、Skills 和消息入口放在一起，让 AI 助手可以在终端、本机界面或远程环境里持续工作。

如果你只是偶尔问 AI 几句话，普通网页聊天工具就够了。Hermes Agent 更适合这几类人：

- 想在终端里让 AI 帮你读项目、看文件、整理信息。
- 想把 AI 助手接到 Telegram、Discord、Slack 等消息入口。
- 想把常用流程沉淀成 Skills，以后反复使用。
- 想自己选择不同模型供应商，而不是只绑定一个入口。
- 想让 Agent 长期运行，逐步接入自动化任务。

官方 README 提到，Hermes Agent 支持 Nous Portal、OpenRouter、NVIDIA NIM、Kimi、OpenAI、自定义端点等多种模型或服务。简单说，它更像一个 Agent 底座，而不是某个固定模型的客户端。

## Hermes Agent 和 Codex 有什么不同

很多人会把 Hermes Agent 和 Codex 放在一起比较，因为它们都能在终端里工作，也都能读取项目和调用工具。但它们的侧重点不一样。

Codex 更像工程协作者，重点是围绕代码仓库完成任务。比如理解项目、修改文件、运行检查、写测试、修 bug、做代码审查，以及把结果变成可以验收的改动。

Hermes Agent 更像个人或团队的 AI 工作台，重点是把模型、工具、记忆、Skills、消息入口和长期任务串起来。它也可以做代码相关工作，但更大的价值在于把 Agent 放进你的日常流程里。

![Hermes Agent 与 Codex 中文差异速览](<../08-素材库/图片/正文插图/29-Hermes Agent 新手快速上手：从安装到跑通第一个 Agent-正文插图02.png>)

我的建议是这样选：

- 如果任务主要发生在代码仓库里，优先用 Codex。
- 如果任务需要长期运行、接消息平台、接不同工具或沉淀个人助手，Hermes Agent 更值得研究。
- 如果你两者都用，可以让 Codex 负责工程交付，让 Hermes Agent 负责跨工具、跨入口的助手流程。

这样理解会更轻松：Codex 像一个会认真改代码的同事，Hermes Agent 像一个可以慢慢配置和训练的 AI 工作台。

## Hermes Agent 的优势在哪里

Hermes Agent 最吸引我的地方，不是某一个功能特别神，而是它把 Agent 需要的几块能力放在了一起。

第一是模型入口比较灵活。你可以根据成本、速度、可用性和能力选择不同 Provider，也可以接自定义端点。对长期使用的人来说，这能减少被单一入口卡住的风险。

第二是工具能力比较完整。它提供 Tool Gateway，可以接搜索、浏览、图片、语音、文件等能力。新手不用第一天全部打开，但知道它后面能扩展到这些方向，会更容易规划。

第三是 Skills 值得长期沉淀。你可以把反复使用的流程整理成 Skill。比如每次分析一个新项目，都先读 README、找启动命令、找测试命令、列风险清单。一次写好，以后重复调用。

第四是它支持 MCP。MCP 可以理解成让 Agent 接外部工具的一套接口。比如你希望 Agent 访问数据库、知识库、内部系统或更多开发工具，就会用到这类能力。

第五是它可以继续往消息入口和定时任务扩展。等本机用顺了，再接 Gateway、Cron 或远程运行，才更安全。

![Hermes Agent 中文工具能力地图](<../08-素材库/图片/正文插图/29-Hermes Agent 新手快速上手：从安装到跑通第一个 Agent-正文插图03.png>)

新手不需要一次吃透这些概念。你只要先记住一句话：先跑通对话，再加工具；先只读，再执行；先本机，再远程。

## 安装前准备

正式安装前，建议先准备三件事。

第一，确认你有一个可用的模型入口。官方 Quickstart 给出了 Nous Portal 设置方式，也支持其他 Provider。新手不要一开始就配多个入口，先选一个你能稳定使用的。

第二，确认本机 Node 环境可用。Hermes Agent 的 Quickstart 使用 `npx @nousresearch/hermes-agent` 启动，至少要保证终端里能正常运行 Node 相关命令。

第三，先找一个低风险目录练习。不要一上来就在重要生产项目里测试。可以新建一个空目录，或者用一个你熟悉的学习项目。

## 第一步：安装 Hermes Agent

官方 Quickstart 给出的方式很直接，打开终端后运行 `npx @nousresearch/hermes-agent`。

第一次运行时，你的目标不是配置所有高级功能，而是确认命令能启动、界面能打开、基本交互正常。

如果这一步报错，先不要继续配置 Gateway、MCP 或 Skills。优先检查 Node 环境、网络访问、包管理器缓存和命令是否写对。

## 第二步：选择模型和 Provider

Hermes Agent 启动后，需要先告诉它使用哪个模型入口。官方文档中提到，可以通过 `hermes model` 或 `hermes setup --portal` 进行配置。

新手这里最容易贪多。我的建议是只选一个 Provider，跑通以后再考虑备用模型。

你只需要确认四件事：

- Provider 能连接。
- 模型名称没有写错。
- 账号或密钥有权限。
- 发一条简单消息后能得到正常回复。

这一步没稳定之前，不要配置工具。因为模型链路不通时，后面所有错误都会变复杂。

## 第三步：跑通第一次对话

第一次对话不要让 Agent 做复杂任务。你可以让它完成一个很容易检查的动作。

例如在一个项目目录里，可以把任务写成这样：

> 请先只读当前目录，告诉我这里可能是什么项目，主要入口在哪里，需要哪些运行命令，不要修改任何文件。

这个任务足够简单，也足够有用。你可以马上检查它说的文件是否真的存在，命令是否来自 README 或配置文件，而不是凭空猜测。

如果它能把目录、文件和命令说清楚，说明基本链路已经跑通。

## 第四步：验证会话能恢复

能回答一次，不代表能长期使用。下一步要验证会话恢复。

你可以用 `hermes --continue` 或 `hermes -c`，让它接着上一轮对话继续工作。然后再给它一个明确的小任务：

> 基于刚才读到的信息，帮我整理一份 8 条以内的新手上手清单。

如果它能接住上下文，并且没有把前面内容忘掉，这个最小闭环就基本成立了。

## 第五步：再打开 Skills、工具和 MCP

前面都跑稳以后，再考虑扩展能力。

Skills 适合沉淀固定流程。比如项目上手、资料整理、代码审查、会议纪要、客服回复，都可以做成一套稳定步骤。

工具和 MCP 适合连接外部能力。但只要涉及读写文件、执行命令、联网搜索、访问账号或发送消息，就要先想清楚权限边界。

对新手来说，最稳的顺序是：先用只读工具，再用写入工具；先在本机练习，再接远程服务；先自己用，再开放给团队。

## 第六步：需要手机或团队入口时，再配置 Gateway

Gateway 的价值是把 Agent 接到 Telegram、Discord、Slack 等消息平台。这样你不用每次打开终端，也能在聊天软件里触发任务。

但它不适合作为第一步。消息平台会带来更多配置项，比如 token、白名单、回调、身份权限和消息触发规则。CLI 还没跑通之前，直接接 Gateway 只会增加排错难度。

更稳的做法是：先本机 CLI，后 Web UI，再 Gateway。每加一层入口，都确认一次权限和验收标准。

## 一套适合新手的 30 分钟练习

如果你想今天就上手，可以按下面这套节奏走。

前 10 分钟，只做安装和模型配置。先确认 `hermes` 命令可用，再选择一个模型入口，发一条简单消息。

第 10 到 20 分钟，在一个低风险项目里让它只读目录，输出项目类型、入口文件、运行命令和测试命令。不要让它修改文件。

第 20 到 30 分钟，用 `hermes -c` 恢复会话，再让它把刚才的信息整理成新手上手清单。如果清单能被你验证，这次练习就算合格。

## 可复用的新手任务卡

你可以直接复制下面这段，用在自己的项目里：

> 请先只读当前项目，不要修改任何文件。按顺序完成：1. 判断项目类型和主要语言；2. 找出最可能的启动入口；3. 找出测试、构建或检查命令；4. 说明哪些结论来自文件证据，哪些只是推断；5. 最后给我一份 8 条以内的新手上手清单。

等这张任务卡跑通后，再换成稍微复杂一点的版本：

> 请根据当前项目的 README 和配置文件，为一个新成员写一份 30 分钟上手路线。要求包含环境准备、首次启动、验证命令、常见错误和不要做的危险操作。先只输出计划，不执行命令。

这类任务很适合训练你和 Agent 协作，因为它结果清楚、风险低，也方便你判断 Agent 有没有胡说。

## 最后给新手的使用建议

Hermes Agent 的价值，不在于第一天把所有功能都打开，而在于你能把它一点点变成自己的 AI 工作台。

第一天只追求三件事：能安装、能回复、能恢复会话。第二天再加低风险工具和 Skills。等你能看懂它为什么调用工具、输出是否可验证，再接入 Gateway、Cron、MCP 和团队入口。

按这个顺序来，Hermes Agent 就不会是一堆复杂功能名，而会变成一条很清楚的上手路线：先会对话，再会使用工具，最后再成为可以长期运行的 AI 助手。
