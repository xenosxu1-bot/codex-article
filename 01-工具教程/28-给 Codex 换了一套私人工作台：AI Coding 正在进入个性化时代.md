> 一句话结论：Dream Skin 能给 Codex 加上自己的背景，但它是第三方工具。把安装、任务页验证和一键还原都跑通，再把它当作日常工作台的一部分。

Dream Skin 不是 OpenAI 官方功能，而是通过本机调试接口给 Codex 叠加主题。本文按 Windows 的 Microsoft Store 版 Codex 实测整理。仓库的 Windows 目录主要由安装、启动、验证和还原四类脚本组成；截至 **目前**，GitHub 页面显示它有 **11,684 Star、1,180 Fork**。热度不等于安全性，开始前先保存未发送的输入。

## 用 Codex 自动安装，比手敲更稳

在一个空文件夹打开 Codex，先把下面这段需求交给它。重点不是让它立刻执行，而是先让它读脚本、确认路径和改动范围。

> 请在当前工作区安装 GitHub 项目 Fei-Away/Codex-Dream-Skin。先克隆到 tools 目录，检查 Node.js 是否为 22 或更高版本，并检查 Microsoft Store 版 Codex 是否已安装。阅读仓库 windows 目录中的安装脚本，告诉我实际入口文件、会修改哪些本地配置、如何还原。不要修改 WindowsApps，也不要执行安装，等我确认。

确认说明无误后，再发第二段：

> 继续执行安装，执行前请提醒我保存未发送的内容；安装后启动 Dream Skin，打开一个新任务页做验证，确认侧栏、建议卡、输入框和滚动都正常，再告诉我验证结果与还原命令。

本次实测环境要求 Node.js 22 或更高。克隆完成后，不要假定脚本一定在同一个子目录：用 Codex 或文件搜索确认 `install-dream-skin.ps1` 的实际位置。本次测试版本的入口文件位于 `windows` 根目录。

## 设置一张自己喜欢的皮肤

安装后，在系统托盘打开 **Codex Dream Skin - Tray**：先保存当前主题，再选择更换背景图，导入 PNG、JPEG 或 WebP。首次选图建议用浅色模式，尤其是白底插画；深色模式下，浅色图容易让输入区和次级文字对比不足。

背景图只管视觉，不会改变 Codex 的模型、权限或任务能力。选图时遵守三条就够了：

- 不用带软件界面、按钮和文字的截图。
- 把主体留在输入框和左侧导航之外。
- 只使用你有权使用的图片，并保留原图和授权记录。

## 30 秒验证：能做任务才算装好

新建一个空任务，不要只看首页背景。依次确认：能输入文字、侧栏可点击、建议卡可用、页面能滚动且没有横向溢出。项目自带验证脚本时，优先让 Codex 找到后执行；本次测试的命令为：

> powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\verify-dream-skin.ps1

矿工猫图在浅色模式下通过了新任务页的结构检查；深色模式虽然能加载，但这张白底图的对比度不适合长期阅读。换深色皮肤时，优先选暗底图片，再重新做一次新任务页检查。

## 不喜欢了，恢复默认 Codex

恢复前先保存工作。让 Codex 在克隆目录中定位 `restore-dream-skin.ps1` 并解释将要执行的动作；本次实测的恢复命令为：

> powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\restore-dream-skin.ps1 -RestoreBaseTheme -PromptRestart

它会提示重启 Codex。重启后新建空任务，确认默认界面、输入框和侧栏正常即可。

最后留住两个习惯：Codex 或 Dream Skin 更新后重新验证；不要手工修改 `WindowsApps`，也不要把本机调试端口暴露到局域网。这样既能保留个人工作台的趣味，也能随时退回默认界面。
